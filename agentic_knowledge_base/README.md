# Agentic Knowledge Base

This directory contains a comprehensive, machine-readable knowledge base generated from the entire Financial Learning Hub repository. It is specifically designed to be portable and easily consumed by Large Language Models (LLMs), agentic workflows, and human-in-the-loop systems to develop a deep and instant understanding of credit analysis, financial modeling, wealth management, and sustainable finance.

## Generation Scripts

The files in this directory were generated by running:

1.  `python build_kb.py`: Scans the entire repository (excluding build directories and old KB folders), extracting content from Markdown, JSON, and Text files, as well as parsing glossaries and decision trees from `Toolkits_and_Checklists/checklists_registry.json`.
2.  `python generate_ttl.py`: Reads the generated `knowledge_graph.json` and uses the `rdflib` library to output a standard RDF Turtle format (`.ttl`) graph of the knowledge.

To regenerate these files after adding new content to the repository, simply run these two scripts again from the root of the repository.

## Available Artifacts

### 1. `knowledge_base.jsonl` (JSON Lines format)

*   **Purpose:** The core textual knowledge base, ideal for Retrieval-Augmented Generation (RAG) systems.
*   **Structure:** Each line is a valid JSON object representing a "chunk" of knowledge.
*   **Contents:** Includes the file source, a list of header "breadcrumbs" for context, the text chunk itself, the type of content (e.g., text, perspective, definition, data), and length.
*   **Usage:** Load this into a vector database (e.g., Pinecone, Weaviate, Milvus) to perform semantic search, enabling agents to retrieve the most relevant financial concepts when answering user queries.

### 2. `knowledge_graph.json` (Graph format)

*   **Purpose:** A structural representation of how concepts, files, and text chunks relate to each other.
*   **Structure:** Contains a list of `nodes` (Files, Chunks, Concepts) and `edges` (CONTAINS, MENTIONS).
*   **Usage:** Can be used by agents to traverse relationships (e.g., finding all chunks that mention a specific glossary term, or finding all concepts mentioned in a specific primer).

### 3. `import.cypher` (Neo4j Graph Database script)

*   **Purpose:** Ready-to-use commands to import the knowledge graph into a Neo4j database.
*   **Usage:** Run this script in the Neo4j browser or via the `cypher-shell` to instantiate a powerful graph database. This allows agents to use Cypher queries to perform complex analytical queries (e.g., "Find the shortest path between 'Credit Default Swap' and 'Sustainable Finance'").

### 4. `knowledge_triples.ttl` and `ontology.ttl` (RDF Ontologies)

*   **Purpose:** Standard Semantic Web representations of the knowledge.
*   **`ontology.ttl`:** Defines the classes (File, KnowledgeChunk, Concept) and properties (contains, mentions).
*   **`knowledge_triples.ttl`:** Contains the actual data nodes and their relationships in Turtle format.
*   **Usage:** Can be loaded into triple stores (like GraphDB or Apache Jena) and queried using SPARQL. This is excellent for ensuring logical consistency and integrating with other external financial ontologies.

### 5. `decision_trees.json` (Structured Logic)

*   **Purpose:** Captures the logic of checklists and step-by-step procedures (like the Analyst Workbench checklists) into a machine-traversable format.
*   **Usage:** Agents can use this file to walk users through a multi-step credit analysis process, asking the user questions at each node and providing context based on the current step in the tree.

### 6. `system_prompts.md` & `prompts.json` (Prompt Engineering)

*   **Purpose:** Useful prompt templates designed specifically to interact with this knowledge base.
*   **Usage:** Provides best practices and starting points for framing LLM interactions, ensuring the LLM acts as an expert financial analyst.

## Integration Examples

*   **RAG Agent:** Load `knowledge_base.jsonl` into a vector store. When a user asks "What are the key covenants in a broadly syndicated loan?", the agent retrieves the top 5 chunks matching the query and uses them to construct an informed answer.
*   **Interactive Tutor:** Use `decision_trees.json` to guide a user through a "Credit Rating Scorecard" checklist. The agent presents the first decision node, waits for the user's input, and then proceeds to the next relevant node.
*   **Graph Explorer:** Import `import.cypher` into Neo4j. The agent can take a complex user question, translate it into a Cypher query, execute it against the database, and return a synthesized summary of the interconnected concepts.
